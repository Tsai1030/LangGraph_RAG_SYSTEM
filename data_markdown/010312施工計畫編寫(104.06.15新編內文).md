# 010312 施工計畫編寫

## 文件中繼資料

- 文件名稱：施工計畫編寫
- 作業流程節次：010312
- 版本日期：104.06.15 新編
- 來源類型：中華工程股份有限公司工務所管理作業手冊

## 1. 目的

為強化公司公共、民間、統包、BOT 工程及投資案競爭力，對提案前可行性評估、設計檢核、價值工程提升、關鍵技術運用、備標階段服務建議書與開工後施工計畫書編撰，透過最佳內容、流程及標準化製作架構，使施工規劃與執行者更容易形成共識並快速理解，以謀取最大利益。

得標後，工務所得按最經濟、可行、完善之標準化作業提報施工計畫，儘速動員開工，並將各分項計畫作業製表條列訂定完成期限追蹤辦理。

## 2. 定義及範圍

### 2.1 定義

| 編號 | 名稱 | 說明 |
|---|---|---|
| 2.1.1 | 備標階段主辦單位 | 業務單位為工作執行之權責單位。 |
| 2.1.2 | 得標階段主辦單位 | 指派工務所為工作執行之權責單位。 |
| 2.1.3 | 備標階段協辦單位 | 備標團隊工作執行之支援單位。 |
| 2.1.4 | 得標階段協辦單位 | 公司後勤部門工作執行之支援單位。 |

### 2.2 範圍

凡備標階段及得標工程在契約規定期程內，應提送之相關計畫工作事宜均屬之。

## 3. 流程及作業說明

### 3.1 流程總覽

## 流程摘要
1. 備標任務交付
2. 工法、工序、風險評估
3. 服務建議書製作及彙整
4. 投標
5. 支援施工計畫製作
6. 整體施工計畫製作綱要
7. 分項施工計畫製作綱要
8. 設施工程分項施工計畫製作綱要
9. 施工中查核與檢討
10. 結案報告資料收集
11. 知識庫建檔
12. 結案

## 作業流程敘述
* 備標任務交付後，先進行工法、工序及風險評估。
* 完成評估後，進行服務建議書製作及彙整。
* 服務建議書完成後，進入投標階段。
* 若未得標，流程回饋至知識庫建檔，作為後續案件經驗累積。
* 若得標，則進入支援施工計畫製作階段。
* 支援施工計畫製作後，依序展開整體施工計畫製作綱要、分項施工計畫製作綱要及設施工程分項施工計畫製作綱要。
* 施工計畫執行期間，持續進行施工中查核與檢討。
* 施工中查核與檢討後，進行結案報告資料收集。
* 結案報告資料收集完成後，辦理知識庫建檔。
* 知識庫建檔完成後，辦理結案。

## 節點說明
* 備標任務交付：啟動施工計畫相關前置作業。
* 工法、工序、風險評估：分析施工方法、施工順序及可能風險。
* 服務建議書製作及彙整：彙整投標所需技術與服務內容。
* 投標：進行投標作業，並判定是否得標。
* 支援施工計畫製作：得標後開始支援後續施工計畫文件編製。
* 整體施工計畫製作綱要：建立整體施工計畫架構。
* 分項施工計畫製作綱要：依工程分項編寫施工計畫內容。
* 設施工程分項施工計畫製作綱要：補充設施工程相關之分項施工計畫。
* 施工中查核與檢討：施工期間進行執行情形查核及改善檢討。
* 結案報告資料收集：彙整結案所需資料與成果。
* 知識庫建檔：將執行經驗、成果及檢討事項納入知識庫。
* 結案：完成案件收尾作業。

### 3.2 備標階段

#### 3.2.1 備標任務交付

1. 依公司策略及業務需求，選定標案，召開啟始會議交付備標任務。
2. 配合標案先期可行評估、設計階段機會協商（價值工程導入）、招標階段公開閱覽、釋疑等協助工作。
3. 由業務單位成立備標團隊，指派專案負責人。

#### 3.2.2 工法、工序、風險評估

1. 依標案類型，各備標團隊指派領域專才人員協助評估作業，並作為日後得標工程施工及計畫編製依據。
2. 工法、工序須符合規範、品質無虞、正確、安全、經濟且具可執行性。
3. 潛在風險需列明風險項目及因應對策，並正確量化風險成本。
4. 應填列於標案風險評估表。

#### 3.2.3 標案風險評估表重點欄位

| 欄位 | 說明 |
|---|---|
| 標案名稱 | 評估標的名稱 |
| 標案編號 | 專案代碼 |
| 業主 | 招標機關或業主單位 |
| 標案位置 | 工程位置 |
| 標案類型 | 例如港灣、建築、土木等 |
| 預算金額 | 預估或公告預算 |
| 決標方式 | 例如最低標、最有利標 |
| 風險強度 | 高度 / 中度 / 一般 / 低度 |
| 風險結果 | Go / Go with conditions / No Go |
| 建議 / 解決 / 說明 | 對各風險項目之因應說明 |

---

### 3.3 服務建議書製作及彙整

1. 依標案需求製作服務建議書，可分為：
   - 自行撰寫
   - 委託專業設計（顧問）公司共同製作
2. 自行撰寫時，採公司任務部門協同原則，依招標單位規定格式、項目及需求編製，並考量得標後施工計畫製作之延續性與可行性。
3. 委外共同製作時，須經評估與核准後辦理，由本單位協助設計及價值工程內容檢核。
4. 最終版本由業務單位核定後，於招標單位規定期限內送達。
5. 服務建議書格式以招標單位規定為準。

---

### 3.4 投標後與得標執行階段

#### 3.4.1 投標結果分流

| 狀態 | 後續作業 |
|---|---|
| 得標 | 支援施工計畫製作 |
| 未得標 | 知識庫建檔後結案 |

#### 3.4.2 支援施工計畫製作

1. 參考備標階段各項評估編製，由工務所指派專人負責主編，各功能單位協助辦理。
2. 應於契約規定期限內提送。
3. 整體施工計畫原則需於開工後 30 日完成。
4. 分項施工計畫需依契約規定分階段提送。
5. 各類型施工計畫編制可參考知識庫範例。

---

### 3.5 整體施工計畫製作綱要

1. 公共工程部份：依契約或公共工程委員會相關規定編製。
2. 製作綱要架構與主要內容，至少應有十五章綱要，視契約要求增減。
3. 民間工程主要依契約規定編製，亦可參酌公共工程委員會相關製作綱要架構。
4. 各功能單位需協助辦理。
5. 公共工程整體施工計畫製作綱要目錄可參考附件一範例。

---

### 3.6 分項施工計畫製作綱要

1. 分項施工計畫之目的，在於配合整體施工計畫，完成特定施工項目之技術性指導功能。
2. 常見項目如：
   - 基樁工程
   - 鋼筋工程
   - 混凝土工程
   - 門窗工程
   - 磁磚工程
3. 應依契約規定或監造單位要求及工程內容檢討。
4. 對送審管制應訂定管制表格，以利控管。
5. 內容應包含：
   - 作業進度表
   - 分項品質計畫
   - 施工要領
   - 品質管理標準
   - 材料及施工檢驗程序
   - 自主檢查表
6. 除上述內容外，尚應包括十一章綱要，視契約要求增減。
7. 可參考附件二、附件三範例。

---

### 3.7 設施工程分項施工計畫製作綱要

適用於工程範圍包含設施或設備類工程時，例如：

| 類別 | 可能提送項目 |
|---|---|
| 設施工程 | 污水處理廠、焚火廠、電業設備工程、機電或設備工程等 |
| 給排水設備工程 | 給排水系統相關計畫 |
| 消防設備工程 | 消防系統相關計畫 |
| 電力設備工程 | 電力系統相關計畫 |
| 空調設備工程 | 空調系統相關計畫 |
| 監控設備工程 | 監控系統相關計畫 |

補充原則：
1. 設施工程分項施工計畫綱要原則依 6.2 項編製。
2. 施工界面整合應針對：
   - 設施與設備間界面
   - 設備與土建間界面
   - 單項設備性能測試界面
   - 多設備起動測試界面
   - 試運轉界面
3. 整合過程須考量工作先後順序與系統權重。
4. 整合成果應盡量以施工圖呈現。

---

### 3.8 施工中查核與檢討

1. 本單位需主動巡視協助在建工程。
2. 就工區現況提出有效、精進方案及潛在風險項目。
3. 應簽報工務部門及決策主管，以達處置時效。
4. 可形成評估報告、風險說明、因應對策與管控表等成果。

---

### 3.9 竣工、資料收集、知識庫建檔與結案

#### 3.9.1 結案報告

1. 施工中廣泛收集各技術文件、計畫書，製作成教育訓練教材。
2. 竣工階段協助工務所彙整相關資料，編製竣工報告書並建檔，以充實知識庫。

#### 3.9.2 資料收集與知識庫建檔

1. 資料收集與檢討完成後，知識庫建檔。
2. 建檔後轉置 PO-ERP 專案管理系統。

#### 3.9.3 結案

依規定完成相關結案待辦事項後結案。
